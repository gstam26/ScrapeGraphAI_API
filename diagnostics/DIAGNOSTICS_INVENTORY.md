## Best diagnostic to run next

Run `diagnostics/acquire_report.py` next:

```powershell
python diagnostics/acquire_report.py samples/input1.xlsx --backend firecrawl --output outputs/input1_acquire_report.xlsx
```

It is the single best current diagnostic for evaluating whether crawl selection is the bottleneck because it shows both sides of the acquire decision: pages actually fetched (`Pages`) and candidate URLs that were scored, followed, or rejected (`Crawl Candidates`). If those sheets show relevant candidates being rejected or budget spent on low-value pages, crawl selection is the problem. If crawl selection looks reasonable but facts are still missing, follow with `extract_report.py` and then `aggregation_trace.py`.
